# @poseui/on

Typed DOM event registration. Declare your targets and handlers up front, activate them against a DOM subtree when you're ready, and clean up with a single function call.

No dependencies. No framework required. Works with any HTML — including markup generated by [pose](https://github.com/guarana-studio/poseui).

```ts
import { createEventMap } from "@poseui/on";

const events = createEventMap();

events.target<HTMLButtonElement>("#submit").on("click", (e) => {
  e.currentTarget.disabled = true; // e.currentTarget is HTMLButtonElement — no cast
});

events.target<HTMLInputElement>(".search").on("input", (e) => console.log(e.currentTarget.value));

events
  .targets<HTMLLIElement>("ul li")
  .on("mouseenter", (e) => e.currentTarget.classList.add("hovered"))
  .on("mouseleave", (e) => e.currentTarget.classList.remove("hovered"));

const unmount = events.mount(document.querySelector("#app"));

// later — removes every listener this instance attached:
unmount();
```

## Core concepts

**`createEventMap()`** — creates an isolated instance with its own registry. No global state, no shared side effects between instances. Create one per page section, component, or dynamic region.

**`.target<T>(selector)`** — register a single-element target. Uses `querySelector` against the mount root — matches the first element. The generic `T` types every `.on()` call on this handle: `e.currentTarget` will be `T`, and the event type will be drawn from `T`'s own event map.

**`.targets<T>(selector)`** — same as `.target()` but uses `querySelectorAll`. Every matched element gets the same listeners attached.

**`.on(type, listener)`** — register a listener on the handle. Chainable. Nothing touches the DOM until `mount()` is called — safe to call before your HTML exists.

**`.off(type, listener)`** — remove a previously registered listener, before or between mount cycles. Uses reference equality, same as `removeEventListener`.

**`events.mount(root?)`** — queries all registered selectors within `root`, attaches every accumulated listener, and returns a cleanup function. `root` defaults to `document`. Call this once your HTML is in the DOM.

**`unmount()`** — the function returned by `mount()`. Removes every listener this mount call attached. Idempotent — safe to call multiple times.

## Scoping

Listeners are scoped to the root you pass to `mount()`. Elements outside that root are never matched, even if they share the same selector:

```ts
const events = createEventMap();
events.target<HTMLButtonElement>(".btn").on("click", handler);

// Only attaches to .btn elements inside #sidebar — not the one in #header
const unmount = events.mount(document.querySelector("#sidebar"));
```

## Dynamic content

Register your handlers once, then mount and unmount as content comes and goes:

```ts
async function openModal() {
  const events = createEventMap();

  events.target<HTMLButtonElement>(".modal-close").on("click", closeModal);
  events.target<HTMLFormElement>(".modal-form").on("submit", handleSubmit);

  document.body.insertAdjacentHTML("beforeend", await fetchModalHTML());

  const unmount = events.mount(document.body.lastElementChild!);

  return unmount; // caller tears down when modal closes
}
```

## Multiple mount roots

The same `EventMap` instance can be mounted against multiple roots independently. Each `mount()` call returns its own cleanup:

```ts
const events = createEventMap();
events.targets<HTMLButtonElement>(".delete-btn").on("click", handleDelete);

const unmountList = events.mount(document.querySelector("#list"));
const unmountSidebar = events.mount(document.querySelector("#sidebar"));

// Tear down independently:
unmountList(); // list listeners removed, sidebar listeners intact
unmountSidebar();
```

## Deferred registration

Targets and listeners are accumulated in memory — no DOM access happens until `mount()`. This means you can define your entire event map at module load time, before any HTML is rendered:

```ts
// Defined at import time — no DOM access yet
const events = createEventMap();
events.target<HTMLButtonElement>("#save").on("click", save);
events.target<HTMLButtonElement>("#discard").on("click", discard);

// Activated once the page is ready
document.addEventListener("DOMContentLoaded", () => {
  events.mount();
});
```

## A note on duplicate listeners

Registering the same function reference for the same event type twice on the same handle calls `addEventListener` twice with the same arguments. The browser deduplicates identical `(element, type, fn)` triples — so the handler fires once, not twice. This matches native `addEventListener` behaviour. If you need separate firings, use separate function references.
